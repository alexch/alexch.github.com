We're definitely doing a mobile webapp, and doing it first. But in order to get into the app store -- and to use some APIs like push notifications (the poor man's background tasks) and GPS and acceleration -- we'll need to make a native app. And if we want the native app to work offline, we'll need to provide some of the logic around the things I mentioned like score recalculation and command descriptions (we've got a few dozen commands the user can perform, and most of their descriptions are made algorithmically -- "[User] added [User] as a follower to [Task]") for use during the period of time between syncs. We've already got that code in Ruby. MacRuby allows us to compile Ruby into LLVM bytecode, so we are very close to the cross-layer code ideal -- an ideal that people keep forgetting about, since they're so used to having one language for the server and a different one for the client these days. But I forget nothing!
